Il Game Engine è indipendente da UI e Database.
Il Domain Model non contiene SQL.
Ogni Service ha una sola responsabilità.
Gli Eventi sono dati, non codice.
I Nodi rappresentano comportamenti, gli Eventi rappresentano contenuti.
Le Decisioni di Game Design precedono le Decisioni Tecniche.
La persistenza non definisce il dominio.
Ogni funzionalità deve superare il Core Loop Test.
Il codice deve essere estensibile senza modificare il motore.
Preferire semplicità e chiarezza a soluzioni "intelligenti" ma complesse.
